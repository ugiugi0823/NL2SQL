전체 플랜은 다음과 같은 흐름으로 진행됩니다. 먼저, 다룰 10개 도메인을 정하고 각 도메인마다 하나의 테이블 구조를 LLM 프롬프트를 통해 자동 생성합니다. 이때 도메인 설명과 공통 컬럼, 제약조건 등을 명시하여 일관된 JSON 형태의 스키마를 얻고, 일괄(batch) 생성 후 문제가 있는 도메인만 개별(iterative)로 보완합니다. 다음으로 각 테이블에 대해 총 50개의 자연어 질의를 준비하는데, 난이도 수준을 Basic부터 Expert까지 네 단계로 나누어 비율에 맞춰 배분합니다. 준비된 스키마 JSON과 질의, 난이도 정보를 포함한 프롬프트로 LLM에게 SQL을 생성하게 하고, 생성된 쿼리를 파싱 오류 여부, 의도 적합성(샘플 데이터 결과 비교), 실행 시간, 토큰 소비량 등으로 자동 검증합니다. 검증 결과를 난이도와 도메인별로 집계해 정확도와 성능 지표를 분석한 뒤, 오류 유형에 맞춰 프롬프트의 Few-shot 예시 수나 제약 조건 문구를 보강하는 튜닝 사이클을 반복합니다. 마지막으로 전체 과정을 대시보드와 보고서로 시각화하여 도메인별·난이도별 강약점을 파악하고, 최종적으로 가장 효과적인 프롬프트 설정을 확정하는 방식입니다.
